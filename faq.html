Frequently/Potentially Asked Questions

QuickTile complains about WINDOW_CHANGE_X and doesn't work. How do I fix this?

Do the errors you see look like this?

** (quicktile.py:1855): WARNING **: Trying to register gtype 'WnckWindowState' as enum when in fact it is of type 'GFlags'

** (quicktile.py:1855): WARNING **: Trying to register gtype 'WnckWindowActions' as enum when in fact it is of type 'GFlags'

** (quicktile.py:1855): WARNING **: Trying to register gtype 'WnckWindowMoveResizeMask' as enum when in fact it is of type 'GFlags'
[some other stuff]
AttributeError: 'module' object has no attribute 'WINDOW_CHANGE_X'

If so, then your Python bindings for libwnck are broken and you need to report the problem on your distro's issue tracker.

(Specifically, the bindings were built against a copy of libwnck with a different ABI than the one you're trying to use them with.)

While you wait for them to fix it, you should be able to solve the issue yourself by rebuilding the bindings from source.

I set ModMask = Control Shift Mod1 but QuickTile isn't responding. Help?
This usually happens when the Compiz Grid plugin is still consuming keystrokes. Make sure you've disabled the Grid plugin in CCSM and then restart Compiz (eg. log out and back in) to make sure it's let go of the keybindings.
How do I keep QuickTile from positioning windows over/under my panels?

You've got an old ~/.config/quicktile.cfg with a workaround for another problem that no longer exists.

Open it up and change UseWorkarea = False to UseWorkarea = True

QuickTile resizes but doesn't move my windows. What do I do?

Some window managers can be set to ignore requests by applications to move your windows. This used to be a problem before QuickTile started announcing itself to the system as a window-management utility, but it shouldn't be any longer.

If you are still experiencing this, please report it so the cause can be invetigated. While you wait, you can work around it by disabling the misbehaving application protection in your window manager.

Under Metacity, for example, that is accomplished using the following command:

{% highlight bash %}gconftool-2 --set "/apps/metacity/general/disable_workarounds" false --type bool{% endhighlight %}
How do I stop Terminator from animating to a zero-width/height window under Xfce?
This is a bug in Terminator that QuickTile can't work around and, in rare circumstances, can also be seen with Xfwm's more limited built-in tiling support. Report it to Terminator's author.
Does QuickTile run on MacOS X?

It's not a primary target, since I don't have a Mac to test with, but if SpectrWM can run via X11.app, QuickTile isn't out of the question.

libwnck fails to retrieve the active window but a workaround is currently being developed in issue 28 and the xquartz branch.

A list of shareware alternatives with official OSX support is also available on StackOverflow.

QuickTile doesn't meet my needs. What can I do?

Assuming you don't feel like either contributing code or filing a feature request and waiting, the Wikipedia Tiling window manager page does contain a section listing tiling utilities for X11 desktops. (Windows too, for that matter)
