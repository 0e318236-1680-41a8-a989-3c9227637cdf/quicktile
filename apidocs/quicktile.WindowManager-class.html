reposition(cls, win, geom=None, monitor=gtk.gdk.Rectangle(0, 0, 0, 0), keep_maximize=False, gravity=<enum WNCK_WINDOW_GRAVITY_NORTHWEST of type WnckWindowGravity>, geometry_mask=<flags WNCK_WINDOW_CHANGE_X | WNCK_WINDOW_CHANGE_Y | WNCK_WIND...)
Class Method

source code 

Position and size a window, decorations inclusive, according to the provided target window and monitor geometry rectangles.

If no monitor rectangle is specified, position relative to the desktop as a whole.

Parameters:
  • win (gtk.gdk.Window) - The wnck.Window to operate on.
  • geom (gtk.gdk.Rectangle) - The new geometry for the window. Can be left unspecified if the intent is to move the window to another monitor without repositioning it.
  • monitor (gtk.gdk.Rectangle) - The frame relative to which geom should be interpreted. The whole desktop if unspecified.
  • keep_maximize (bool) - Whether to re-maximize a maximized window after un-maximizing it to move it.
  • gravity (WnckWindowGravity or GDK Gravity Constant) - A constant specifying which point on the window is referred to by the X and Y coordinates in geom.
  • geometry_mask (WnckWindowMoveResizeMask) - A set of flags determining which aspects of the requested geometry should actually be applied to the window. (Allows the same geometry definition to easily be shared between operations like move and resize.)

To Do (1.0.0): Look for a way to accomplish this with a cleaner method signature. This is getting a little hairy. (API-breaking change)
